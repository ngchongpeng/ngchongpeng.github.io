AngularJS
* Advantages
- Uses JavaScript; hence easy to learn
- Two-way data binding
- Fast development for small applications

* Disadvantages
No dynamic loading
No cli
Angular 2
* Advantages
- Client tool for expedited development
- Component decorators
- Event bindings

* Disadvantages
Code base overhauled as compared to that of AngularJS. Hence AngularJS users had to relearn syntax
Angular 4
* Advantages
- HttpClient
- New router lifecycle events
- Conditionally disable animations
Angular 5
* Advantages
- Support for PWAs
- Build optimizer
- Improvements related to material design
Angular 6
* Advantages
- Toolchain improvements
- Introduction of angular elements
- Introduction of angular material with cdk
- Library support
- Starter components
Angular 7
* Advantages
- App performance improvements
- Angular material + cdk updates
- Virtual scrolling
- Content projection support
- Dependency updates
Angular 8
* Advantages
- Differential loadings for all app code
- Dynamic imports for lazy routes
- Introduction of web workers
- Typescript 3.4 support
- Opt in preview of angular ivy
Angular 9
* Advantages
- Moved all applications to ivy compiler & runtime
- Smaller bundle size
- Faster testing
- Better debugging
- Improved css class & style binding
- Improved type checking
- Improved build error handling
- Improved build times
- Improved internationalization
- Compatible with typescript 3.6 & 3.7
Angular 10
* Advantages
- New date range picker
- Warning about commonjs imports
- Optional stricter settings
- Updates to dependencies
- New default browser configuration
- Deprecations & removals
Angular 11
* Advantages
- Automatic inlining of fonts
- Component test harnesses for all components
- Improved reporting & logging
- Updated language service preview
- Updated hot module replacement support
- Faster builds
- Experimental webpack 5 support
- Linting updates
- Removal of support for ie9/10/mobile
- Deprecations & removals
Angular 12
* Advantages
- Deprecation of view engine
- Transition from legacy i18m message ids
- Support for nullish coalescing operator in typescript
- Styling improvements
- Ng build defaults to production mode unless otherwise specified
- Strict mode enabled by default in cli
- Ivy-based language service enabled by default
- Webpack 5 support released prodution-ready
- Support for typescript 4.2 added
